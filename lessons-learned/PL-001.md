## Solutions

### Unit-testing D1 schema without a Cloudflare binding
D1 and `bun:sqlite` share the SQLite dialect. Apply the drizzle-kit-generated
migration SQL (split on `--> statement-breakpoint`) into an in-memory
`bun:sqlite` `Database`, then wrap it with `drizzle-orm/bun-sqlite` and the same
`schema`. This exercises the real tables/FKs/defaults with zero new infra. Tests
run via `bun test` (built-in), typed with `@types/bun` + a `/// <reference
types="bun" />` at the top of the test file.

### drizzle-kit generate needs no live DB
`drizzle-kit generate` reads the schema and emits migration SQL offline (only
`push`/`migrate` need credentials). The first migration captures auth tables too
since there was no prior baseline — that's correct and the auth source is
unchanged.

## Errors

### oxlint pre-commit blocks SQL/JSON-only commits
The lefthook pre-commit runs `oxlint --fix {staged_files}`; when no JS/TS files
are staged (e.g. the migration commit was only `.sql` + meta `.json`), oxlint
exits 1 with "No files found to lint" and aborts the commit. Use
`git commit --no-verify` for those commits (matches project-memory note).
