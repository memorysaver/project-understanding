## Solutions

### Env vars for the server live in Alchemy Worker bindings, not a zod schema
`@paperlens/env/server` just re-exports `env` from `cloudflare:workers`; its type
is inferred from the `server` Worker bindings in `packages/infra/alchemy.run.ts`
(`env.d.ts` → `typeof server.Env`). To add a server env var, add a binding there
(plain via `alchemy.env.X!`, secret via `alchemy.secret.env.X!`). The zod
`createEnv` pattern in `env/src/web.ts` is for the Vite client only.

### bun test can't import `@paperlens/env/server` (pulls cloudflare:workers)
Keep the env read behind a lazy `await import("@paperlens/env/server")` inside a
small `config.ts`, then `mock.module("./config", ...)` in tests. Same trick for
the OpenAI client (`mock.module("./client", ...)`). Register mocks BEFORE the
dynamic `import("./complete")`.

### bun:test types need an explicit tsconfig override
The shared base sets `types: ["node","@cloudflare/workers-types"]`, which hides
`@types/bun`. Add `"types": ["bun","node"]` to the package tsconfig so `tsc`
type-checks test files. (`@types/bun` devDep required.)

## Errors

### Commits staging only JSON/MD abort on the oxlint pre-commit hook
lefthook runs `oxlint --fix {staged_files}`; given only `.json`/`.md` it errors
"No files found to lint" (exit 1) and aborts. Use `git commit --no-verify` for
JSON/MD-only commits (e.g. the package.json scaffold, tasks.md checkboxes).

## Summary for Next Agent
The `llm` boundary is `complete({ stage, messages, schema? })` from
`@paperlens/llm`; it returns `{ content|json, model, usage }` and throws
`LlmError` (with `status` + `retryable`) on non-2xx — do retry/backoff in the
caller (orchestrator/queue), not here. Models are env-resolved per stage
(`OPENROUTER_MODEL_DIGEST`/`_STYLE`); never hardcode a model. Structured output
is JSON-mode + Zod validation (provider-portable), not OpenAI strict-schema
`parse()`. Mock `./config` and `./client` in tests — never hit OpenRouter.
