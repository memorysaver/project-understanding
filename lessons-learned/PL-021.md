# Lessons: PL-021 (Unpublish / edit a post)

Module: api
Activity: restyle-publish
Date: 2026-06-25
Story: PL-021

## Solutions

### Mixed Date|SQL drizzle .set() type
Setting `publishedAt` to either a Date or a drizzle `sql\`...\`` expression in the
same `.update(posts).set({...})` breaks a `Partial<typeof posts.$inferInsert>`
annotation (SQL isn't a Date). Cleanest fix: don't pre-annotate the set object —
build it inline in `.set({ ...(cond ? { col: sql\`...\` } : {}) })`; drizzle's
own `.set()` parameter type accepts `SQL` per column.

### published_at invariant without a read-first
`coalesce(${posts.publishedAt}, ${Date.now()})` applied only when status flips to
`published` keeps the PL-006 invariant in one statement: fills a null published_at
on first publish, preserves the original timestamp on republish. No read-then-write.

## Errors

### routeTree.gen.ts must be regenerated for a new route
TanStack Router's `routeTree.gen.ts` is committed but generated by the vite
plugin. A new route file (console/posts.tsx) won't typecheck until the tree is
regenerated. The local dev server was down (empty .env), so I regenerated it
headlessly: `new Generator({ config: await getConfig({target:"react",
autoCodeSplitting:true}, webRoot), root: webRoot }).run()` from
`@tanstack/router-generator`, run with bun from apps/web so module resolution
finds local node_modules.
