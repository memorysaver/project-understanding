## Context

The digestor (`packages/digestor/src/index.ts`) fetches the text to digest via the
injected `fetchArxivFullText`, which prefers the arXiv HTML source (`fullTextUrl`)
and **falls back to the stored abstract** when no HTML source is available or the
fetch fails. PL-030 already detects the fallback case —
`abstractOnly = fullText.trim() === paper.abstract.trim()` — and, when true,
appends `ABSTRACT_ONLY_GUARD` to the system prompt so the LLM does not invent
specifics absent from the abstract. That `abstractOnly` boolean is the signal this
change persists and acts on; it is **not recomputed** here.

The orchestrator (`packages/orchestrator/src/index.ts`) runs the pipeline over a
single queue, one message per (paper, stage). `handleDigest` loads the Paper, runs
the digestor when the Paper is not already at/past `digested`, then enqueues
`style`. Idempotency is **resume-from-stage** (`isAtOrPast` over `STATUS_RANK`): a
redelivered message resumes from the last durable intermediate. `enqueueDiscovery`
and `dispatch` (with `DispatchContext { attempt, maxRetries }`) are the entry and
router. `runOnce` does not exist (removed by PL-018); the synchronous driver is
`runPipelineOnce`, an offline eval/smoke path that drains a local in-memory queue.

The `digests` table (`packages/db/src/schema/paperlens.ts`) carries
`contributions / methods / results / rawJson / model` and has no field recording
whether the digest came from full text or only the abstract. Migrations are
generated by drizzle-kit (`bun run db:generate`, `out: ./src/migrations`) and
applied via the PL-001 harness in tests.

## Goals / Non-Goals

**Goals:**
- Persist `source_kind` (`full_text | abstract`) on each Digest; an abstract-only
  digest is queryable as such.
- The orchestrator never publishes an abstract-only paper as a normal post: it
  defers (re-queue with backoff for re-digest) or, at the deferral budget, publishes
  with an explicit lower-confidence flag.
- A paper that was abstract-only and later gains full text is re-digested from the
  full text before/at publish.
- The defer-vs-flag policy is recorded.

**Non-Goals:**
- No change to how full-text-vs-abstract is **detected** (PL-030 owns that signal).
- No new external scheduler / cron cadence (Layer 2); the backoff reuses the
  existing queue requeue idiom, not a new timer subsystem.
- No change to the `style` / `publish` stages' core behavior beyond carrying the
  flag through; no change to the stylist or publisher modules.
- No PDF binary parsing (still out of scope per the digestor spec).

## Decisions

### `source_kind` column on `digests`
Add `sourceKind: text("source_kind", { enum: ["full_text", "abstract"] }).notNull()`
to the `digests` table, with a `full_text` default so existing rows and the common
path need no special handling. Export a `digestSourceKinds` const + `DigestSourceKind`
type alongside the existing enums (matching the `paperStatuses` / `postStatuses`
pattern). Regenerate the drizzle migration into `packages/db/src/migrations/` via
`bun run db:generate` (do not hand-write SQL); the new migration is purely additive
(one column) and must apply cleanly over the PL-001 baseline.

### Digestor sets `source_kind` from the existing signal
In `run`, the digestor already computes `abstractOnly`. Set
`sourceKind: abstractOnly ? "abstract" : "full_text"` on the inserted Digest row.
This is the only digestor change: the detection, the `ABSTRACT_ONLY_GUARD` prompt
steering, and the transaction shape are untouched.

### Orchestrator defer-with-backoff branch (SETTLED POLICY)
After the digestor produces (or has already produced) the Digest, `handleDigest`
reads the Digest's `source_kind`:

- **`full_text`** → unchanged: enqueue `style` as today.
- **`abstract`** and **within the deferral budget** → **defer**: do **not** enqueue
  `style`. Instead re-enqueue the paper for a later `digest` retry with a backoff,
  reusing the existing queue producer (the same `queue.send` idiom the stages use).
  The backoff bounds re-attempts so a paper is re-digested once `fetchArxivFullText`
  returns more than the abstract (ar5iv typically renders full text within ~2 days).
  The retry count is tracked so the budget is enforceable (carried on the requeued
  message and/or derived from a durable counter — the build picks the minimal
  mechanism consistent with the at-least-once queue).
- **`abstract`** and **deferral budget exhausted** → **bounded fallback (recorded)**:
  publish the paper with an **explicit lower-confidence abstract-only flag** rather
  than as a normal post, and rather than deferring forever. This is the recorded
  fallback so the system never blind-publishes and never defers indefinitely.

### Re-digest from full text on retry
A deferred (abstract-only) paper stays digestable: its resume guard must let the
digestor **re-run** when the prior Digest was abstract-only, so a retry that now
sees full text replaces the abstract-only Digest with a `full_text` one before the
paper proceeds to `style`/`publish`. Concretely, the at/past-`digested` short-circuit
in `handleDigest` is relaxed for an abstract-only Digest: re-run the digestor; if it
now produces `full_text`, advance normally; if still `abstract`, defer again (or
flag at the budget). The single-current-Digest-per-Paper invariant is preserved (the
re-digest overwrites/replaces, it does not accumulate).

### Where the flag lives
The explicit lower-confidence flag the bounded fallback publishes is derived from /
carried by the Digest's persisted `source_kind = abstract` (the post is published
from a paper whose current Digest is abstract-only). The build records the policy in
the requirement so an abstract-only published post is distinguishable from a normal
post — never published blind.

## Risks / Trade-offs

- **At-least-once redelivery vs. the deferral counter.** The defer branch must not
  let a redelivered `digest` reset the backoff counter or fan out a second deferral;
  the retry budget is tracked on the durable intermediate / requeued message so a
  duplicate delivery is idempotent (resume-from-stage, consistent with the existing
  `isAtOrPast` guard). Pinned by a unit test (abstract-only paper defers, does not
  advance to `style`).
- **Re-digest must replace, not duplicate.** Relaxing the at/past-`digested` guard
  for abstract-only Digests risks re-running the digestor for an already-full-text
  paper; the relax is scoped to `source_kind = abstract` only, so a `full_text`
  Digest is never re-run. Preserves the at-most-one-current-Digest invariant.
- **Bounded fallback is a real publish path.** A flagged abstract-only post is still
  published (not dropped), so readers/console see it as lower-confidence rather than
  losing it — the trade-off chosen over deferring forever.

## Dependency APIs (consumed)

- **PL-030 digestor (merged):** the abstract-only detection
  (`abstractOnly = fullText.trim() === paper.abstract.trim()`) and `ABSTRACT_ONLY_GUARD`;
  `fetchArxivFullText` (HTML-preferred, abstract fallback). This change reads the
  `abstractOnly` signal and persists it; it does not change detection.
- **PL-018 orchestrator (merged):** `handleDigest`, `dispatch`
  (`DispatchContext { attempt, maxRetries }`), the `queue.send` requeue idiom,
  `enqueueDiscovery`, the `isAtOrPast` / `STATUS_RANK` resume guard, and
  `runPipelineOnce` (offline driver). `runOnce` was removed — not referenced.
- **PL-001 persistence (merged):** the `digests` table + the drizzle migration
  harness (`bun run db:generate`; tests apply migrations over in-memory `bun:sqlite`).
