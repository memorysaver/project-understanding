# Copy/Tone — Capture Questions

Ask these questions one at a time during `/calibrate capture` for copy-tone.

## Questions

1. **Voice personality:** In one sentence, how should this product "sound"? (e.g., "confident but not cocky, technical but accessible")
2. **Reference products:** Which 2-3 products have a voice you'd want to emulate? Which 1-2 would you avoid?
3. **Heading style:** How formal should headings be? (e.g., "Dashboard" vs "Your Dashboard" vs "Here's what's happening")
4. **Button labels:** What convention? (verb + noun: "Create Project", verb only: "Create", noun only: "New Project")
5. **Error messages:** What pattern? (what went wrong + how to fix it? Just the fix? Apologetic or matter-of-fact?)
6. **Empty states:** Encouraging ("Get started by..."), instructional ("No items yet. Create one."), or minimal ("No items.")?
7. **Technical jargon:** Embrace it (audience expects it) or translate it (make accessible)?
8. **Glossary terms:** List terms that should be used consistently. For each: the canonical term, when to use it, and what to avoid.

## Output Artifact

Write to `calibration/copy-tone.yaml` using schema at `references/schemas/copy-tone-schema.yaml`.
